Hermes 데이터베이스 테이블 목록 및 기능

데이터베이스 개요

이 데이터베이스는 게임 사용자 관리, 사용자 활동 추적, 금융 거래 및 고객 관리 기능을 위한 시스템으로 보입니다. 주요 기능은 사용자 계정 관리, 플레이어 정보 관리, 금융 거래 처리, 광고 및 마케팅 관리 등을 포함합니다.

사용자 관리 테이블

테이블 이름 추정 기능
users 시스템 사용자 계정 관리 테이블. 관리자, 에이전트, 매니저 등 시스템에 접근하는 사용자의 정보와 권한을 저장합니다.
user_groups 사용자 그룹 정보를 관리하는 테이블. 다양한 역할과 권한을 가진 사용자 그룹을 정의합니다.
user_sessions 사용자 로그인 세션 관리 테이블. 사용자의 로그인 상태와 세션 정보를 추적합니다.

게임 플레이어 관리 테이블

테이블 이름 추정 기능
players 게임 플레이어 정보 테이블. 게임 이용자의 기본 정보와 상태를 저장합니다.
player_contacts 플레이어의 연락처 정보를 관리하는 테이블. 전화번호, 이메일 등이 포함될 수 있습니다.
player_agents 플레이어와 에이전트 간의 관계를 관리하는 테이블. 어떤 에이전트가 어떤 플레이어를 담당하는지 정의합니다.
player_comments 플레이어에 대한 주석이나 메모를 저장하는 테이블. 관리자나 에이전트가 플레이어에 대해 남긴 메모를 관리합니다.
player_limits 플레이어의 게임 사용 제한 설정을 관리하는 테이블. 게임 사용 시간, 베팅 한도 등이 포함될 수 있습니다.
player_relations 플레이어 간의 관계를 정의하는 테이블. 친구, 가족, 동료 등의 관계를 추적할 수 있습니다.
player_audits 플레이어 활동에 대한 감사 로그를 저장하는 테이블. 중요한 활동이나 변경 사항을 기록합니다.
player_guilds 플레이어 길드 또는 그룹 정보를 관리하는 테이블. 게임 내 팀이나 조직을 추적합니다.
private_players 특별 관리가 필요한 플레이어 정보를 저장하는 테이블. VIP 또는 제한된 접근 권한이 필요한 플레이어 정보를 관리합니다.
player_adjustments 플레이어 계정에 대한 조정 사항을 기록하는 테이블. 보너스, 페널티 등의 계정 조정을 추적합니다.
player_wallets 플레이어의 가상 지갑 정보를 관리하는 테이블. 게임 내 통화나 자산을 추적합니다.
player_wallet_limits 플레이어 지갑의 사용 제한 설정을 관리하는 테이블. 일일 최대 인출 한도 등을 설정합니다.
player_delegations 플레이어 계정 권한 위임 정보를 관리하는 테이블. 계정 관리 권한을 다른 사용자에게 위임하는 기능을 지원합니다.

고객 관리 테이블

테이블 이름 추정 기능
customers 고객 정보 관리 테이블. 플레이어와는 별도로 관리되는 비즈니스 고객 정보를 저장합니다.
customer_contacts 고객 연락처 정보를 관리하는 테이블. 이메일, 전화번호 등의 연락처 정보를 저장합니다.
customer_details 고객의 상세 정보를 저장하는 테이블. 추가적인 고객 프로필 정보를 관리합니다.
customer_sets 고객 그룹 또는 세트를 관리하는 테이블. 고객을 특정 그룹이나 카테고리로 분류합니다.

게임 관련 테이블

테이블 이름 추정 기능
gamers 게이머 정보를 관리하는 테이블. 플레이어와 다른 특별한 게이머 계정을 추적할 수 있습니다.
game_scores 게임 점수 정보를 저장하는 테이블. 플레이어의 게임 성과나 점수를 기록합니다.
game_attends 게임 참여 정보를 기록하는 테이블. 플레이어의 게임 참여 시간과 빈도를 추적합니다.
score_imports 외부에서 가져온 게임 점수를 관리하는 테이블. 다른 시스템에서 가져온 점수 데이터를 저장합니다.

금융 거래 관련 테이블

테이블 이름 추정 기능
money_accounts 금융 계정 정보를 관리하는 테이블. 사용자 또는 플레이어의 금융 계정 정보를 저장합니다.
money_account_transactions 금융 계정 거래 내역을 기록하는 테이블. 입금, 출금, 이체 등의 거래를 추적합니다.
money_flows 자금 흐름을 추적하는 테이블. 시스템 내 자금 이동을 모니터링합니다.
money_flow_events 자금 흐름 이벤트를 기록하는 테이블. 특정 자금 흐름 패턴이나 이벤트를 추적합니다.
money_orders 금융 주문 또는 거래 주문을 관리하는 테이블. 입금, 출금 요청 등을 추적합니다.
money_order_cards 거래에 사용된 카드 정보를 관리하는 테이블. 결제 카드 정보를 저장합니다.
money_operation_cards 운영에 사용되는 카드 정보를 관리하는 테이블. 회사 운영 자금 관리에 사용되는 카드를 추적합니다.
using_operation_cards 현재 사용 중인 운영 카드를 추적하는 테이블. 운영 카드의 사용 상태를 관리합니다.
money_card_groups 카드 그룹을 관리하는 테이블. 여러 카드를 특정 그룹으로 분류합니다.
favorite_money_cards 사용자의 즐겨찾는 카드 정보를 관리하는 테이블. 자주 사용하는 결제 수단을 저장합니다.
money_receipt_files 거래 영수증 파일을 관리하는 테이블. 거래 증빙 문서나 영수증 파일을 저장합니다.
money_trans_files 거래 관련 파일을 관리하는 테이블. 거래 처리와 관련된 다양한 문서 파일을 저장합니다.
money_fee_reports 수수료 보고서를 관리하는 테이블. 거래 수수료 정보와 보고서를 저장합니다.
money_alerts 금융 거래 관련 알림을 관리하는 테이블. 특정 금융 이벤트에 대한 알림 설정과 기록을 저장합니다.
money_notifications 금융 거래 관련 통지를 관리하는 테이블. 사용자에게 발송된 금융 거래 알림을 추적합니다.

광고 및 마케팅 관련 테이블

테이블 이름 추정 기능
ad_nodes 광고 노드를 관리하는 테이블. 광고 컨텐츠나 광고 게재 위치를 정의합니다.
ad_node_groups 광고 노드 그룹을 관리하는 테이블. 관련 광고 노드를 그룹으로 분류합니다.
ad_contacts 광고 연락처 정보를 관리하는 테이블. 광고주나 마케팅 담당자의 연락처를 저장합니다.
ad_node_changes 광고 노드 변경 이력을 관리하는 테이블. 광고 노드의 수정 사항을 추적합니다.
assigned_ad_nodes 할당된 광고 노드를 관리하는 테이블. 특정 위치나 사용자에게 할당된 광고를 추적합니다.
assign_ad_histories 광고 할당 이력을 관리하는 테이블. 과거 광고 할당 정보를 저장합니다.

프로모션 관련 테이블

테이블 이름 추정 기능
promotions 프로모션 정보를 관리하는 테이블. 마케팅 캠페인이나 특별 이벤트를 정의합니다.
promotion_players 프로모션에 참여하는 플레이어 정보를 관리하는 테이블. 프로모션별 참여자를 추적합니다.

시스템 관리 테이블

테이블 이름 추정 기능
assignments 작업 할당 정보를 관리하는 테이블. 사용자에게 할당된 작업을 추적합니다.
assign_histories 작업 할당 이력을 관리하는 테이블. 과거 작업 할당 정보를 저장합니다.
allowed_ips 허용된 IP 주소를 관리하는 테이블. 시스템 접근이 허용된 IP 주소 목록을 저장합니다.
service_states 서비스 상태 정보를 관리하는 테이블. 다양한 시스템 서비스의 현재 상태를 추적합니다.
abnormal_contacts 비정상적인 연락 시도를 기록하는 테이블. 의심스러운 로그인 시도나 연락을 추적합니다.
adjust_invoices 조정된 청구서를 관리하는 테이블. 청구서 수정이나 조정 내역을 저장합니다.

커뮤니케이션 관련 테이블

테이블 이름 추정 기능
talks 대화 또는 메시지를 관리하는 테이블. 사용자 간 커뮤니케이션을 저장합니다.
talk_phrases 자주 사용되는 대화 문구를 관리하는 테이블. 템플릿 메시지나 자주 사용되는 구문을 저장합니다.

기타 테이블

테이블 이름 추정 기능
dime_channels 채널 정보를 관리하는 테이블. 다양한 통신 또는 배포 채널을 정의합니다.
dime_ranges 범위 값을 관리하는 테이블. 다양한 수치 범위나 한계를 정의합니다.
dime_avoid_players 회피해야 할 플레이어를 관리하는 테이블. 특정 플레이어를 모니터링하거나 제한하는 데 사용됩니다.

이 문서는 데이터베이스 테이블 구조 분석을 기반으로 생성되었습니다. 테이블의 실제 기능은 추정된 것이며 실제 용도와 다를 수 있습니다.
