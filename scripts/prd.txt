# 헤르메스 데이터베이스 분석 프로젝트

## 프로젝트 목표
헤르메스 데이터베이스를 분석하고 관리하기 위한 도구를 개발하고, 데이터베이스 구조와 데이터를 이해하여 의미 있는 인사이트를 도출한다.

## 기능 요구사항

### 1. 데이터베이스 연결 및 관리
- 데이터베이스 연결 상태를 지속적으로 모니터링한다.
- 연결 정보를 안전하게 관리하고 저장한다.
- 연결 로그를 유지하고 문제 발생 시 기록한다.

### 2. 데이터베이스 구조 분석
- 모든 테이블과 관계를 분석하고 문서화한다.
- 테이블 스키마를 추출하고 시각화한다.
- 주요 데이터 필드와 코드 값의 의미를 문서화한다.

### 3. 데이터 쿼리 및 분석
- 특정 조건에 맞는 사용자 그룹을 식별하는 쿼리를 개발한다.
- 이벤트 효과와 사용자 행동을 분석한다.
- 패턴과 추세를 파악하기 위한 통계 분석을 수행한다.

### 4. 보고서 생성
- 분석 결과를 시각적으로 표현하는 보고서를 생성한다.
- HTML 및 CSV 형식으로 결과를 내보낸다.
- 주요 발견 사항과 권장 사항을 문서화한다.

### 5. 메모리 뱅크 통합
- 모든 분석 결과와 문서를 메모리 뱅크에 저장한다.
- MCP를 통해 메모리 뱅크를 관리한다.
- 코드, 쿼리, 결과를 구조화된 방식으로 저장한다.

## 기술적 요구사항
- Node.js 또는 Python을 사용한 데이터베이스 인터페이스
- SQL 쿼리 최적화
- 데이터 시각화 (차트, 그래프)
- 데이터 익스포트 기능 (CSV, JSON)
- 메모리 뱅크 API 통합

## 제약 조건
- 데이터베이스 연결은 안전하게 처리되어야 한다.
- 모든 개인 정보는 적절히 익명화되어야 한다.
- 대규모 쿼리는 데이터베이스 성능에 영향을 최소화해야 한다.
- 분석 결과는 재현 가능해야 한다.

## 타임라인
- 1주차: 데이터베이스 연결 및 스키마 분석
- 2주차: 초기 쿼리 개발 및 데이터 분석
- 3주차: 보고서 생성 및 시각화
- 4주차: 메모리 뱅크 통합 및 문서화

## 성공 지표
- 주요 테이블 구조와 관계의 명확한 문서화
- 의미 있는 사용자 그룹을 식별하는 5개 이상의 쿼리
- 이벤트 효과와 사용자 행동에 대한 3개 이상의 인사이트
- 메모리 뱅크에 체계적으로 저장된 모든 분석 결과와 문서
